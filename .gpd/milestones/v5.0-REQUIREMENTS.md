# Requirements: Hg1223 Instrumented Reproduction and Basin Test

**Defined:** 2026-03-29
**Core Research Question:** Can the first carried `Hg1223` low-`TQ` benchmark-window campaign be made instrumented, stage-localized, and falsifiable enough to establish a real retained-state basin, or to force an honest route downgrade without ambiguity about `vQ`, sample state, or warm handling?

## Primary Requirements

### Instrumented Stage-A Runbook

- [x] **RUN-01**: Convert the carried Stage `A` benchmark-window campaign into a collaborator-facing runbook with the exact `Hg1223` nodes `PQ = 10.1`, `18.9`, and `28.4 GPa` and `TQ = 4.2 K` and `77 K`
- [x] **RUN-02**: Define a mandatory logging schema for `PQ`, `TQ`, `vQ`, sample metadata, elapsed stage times, and every planned or unplanned thermal excursion
- [x] **RUN-03**: Define cryogenic-first handling classes and explicit run invalidation rules so uncontrolled warm handling cannot be misread as route failure

### Failure Localization and Evidence Packaging

- [x] **DIAG-01**: Build a failure-mode map that separates target-state failure, quench-trajectory failure, sample-state failure, retrieval loss, and onset-only overclaiming
- [x] **DIAG-02**: Define the minimum per-run evidence package required for any claimed reproduction, including transport, trace logs, and selected ex-DAC follow-up
- [x] **DIAG-03**: Define which measurements belong to the first success gate and which are deferred until after a clean retained-state hit exists

### Route Gates and Backup Preservation

- [x] **GATE-01**: State explicit keep / hold / downgrade criteria for `Hg1223` after the first instrumented campaign
- [x] **GATE-02**: Preserve the bilayer nickelate backup route with current watch thresholds, while keeping `Hg1223` primary unless the first instrumented campaign fails cleanly
- [x] **GATE-03**: Keep the `149 K` room-temperature gap explicit in every milestone closeout and route memo

### Decision and Validation

- [x] **VALD-01**: Tie every decisive protocol or route claim to the carried primary-source papers and prior milestone artifacts with explicit conditions
- [x] **VALD-02**: Reject false progress from any run or memo that omits `vQ`, hides uncontrolled warm excursions, or treats onset-only evidence as basin proof
- [x] **VALD-03**: End the milestone with a next-step memo that says clearly whether the repo should execute the first instrumented `Hg1223` campaign or shift weight toward the backup route

## Follow-up Requirements

### Extended Work

- **EXTD-01**: Add computational support around sample state, oxygen history, or defect stabilization only after the first instrumented campaign package is complete
- **EXTD-02**: Reassess the backup nickelate route if new ambient-pressure evidence narrows the gap further or `Hg1223` fails under recorded controls
- **EXTD-03**: Promote collaborator-facing experiment planning into manuscript-ready figures or supplemental material if the runbook stabilizes

## Out of Scope

| Topic | Reason |
|-------|--------|
| Claiming a room-temperature finished material | Not supported by the present evidence |
| Treating a planned protocol as an experimental reproduction | This milestone prepares the campaign; it does not run it |
| Device engineering or industrial scale-up | Premature before any basin is proven |
| Restarting blind hydride screening | Previous milestones closed that path negatively |
