# Roadmap: v4.0 Hg1223 Protocol Extraction and Reproducibility

## Overview

Milestone `v3.0` closed with a route hierarchy rather than a new material. `Hg1223`-class pressure-quenched cuprates are now the primary route because they combine the strongest carried ambient-pressure benchmark with the smallest remaining gap to room temperature. This milestone narrows from route ranking to route control: exact protocol extraction, cross-system `PQP` transfer, and an experiment-facing reproducibility campaign.

## Phase Details

### Phase 15: Hg1223 Protocol Extraction and Evidence Ledger

**Goal:** Convert `Hg1223` from a route slogan into an exact protocol, stability, and evidence ledger grounded in the full benchmark paper.
**Depends on:** Phase 14 route recommendation
**Requirements:** `PROT-01`, `PROT-02`, `PROT-03`, `VALD-01`
**Contract Coverage:**
- Advances: `claim-hg1223-protocol`
- Deliverables: `deliv-phase15-ledger`, `deliv-phase15-stability`
- Acceptance tests: `test-hg1223-window-explicit`
- Anchor coverage: `ref-hg1223-quench`, `ref-phase11-note`, `ref-phase14-decision`
- Forbidden proxies: `fp-benchmark-without-protocol`

**Success Criteria**

1. The repo states the exact carried `Hg1223` `PQ/TQ/Tc` window from the full paper
2. Thermal, temporal, structural, and magnetic evidence are carried in one stability ledger
3. Missing variables such as `vQ`, sample-count statistics, and long-term room-temperature durability are called out explicitly
4. The repo emits an evidence-grade note that distinguishes confirmed fields from unresolved ones

**Backtracking trigger:** If the full-paper extraction still leaves `Hg1223` too under-specified to act on, Phase 16 must lower confidence in the cuprate route and widen the analog/control emphasis.

**Plans:** 3/3 plans complete

Plans:
- [x] `15-01-PLAN.md` -- Exact `Hg1223` protocol ledger from the full benchmark paper
- [x] `15-02-PLAN.md` -- `Hg1223` stability, retrieval, and bulk-evidence budget
- [x] `15-03-PLAN.md` -- `Hg1223` evidence-grade memo and remaining control gaps

**Executed outcome (2026-03-29):** `Hg1223` is no longer just "the 151 K cuprate benchmark." The repo now carries a real protocol window: reproducible retained `Tc = 147-151 K` at `PQ = 10.1-28.4 GPa` and `TQ = 4.2 K`, a weaker `139 K` result at `77 K`, stability for at least three days at `77 K`, degradation above roughly `170-200 K`, and a retrieved bulk signal of about `78%` with partial annealing. The main remaining missing variable is the unreported numerical `vQ`.

### Phase 16: PQP Transfer Map and Missing-Control Analysis

**Goal:** Determine which `PQP` controls really transfer from `BST` and `FeSe` into `Hg1223`, and rank the missing control fields that still block replication-grade confidence.
**Depends on:** Phase 15
**Requirements:** `XFER-01`, `XFER-02`, `XFER-03`
**Contract Coverage:**
- Advances: `claim-pqp-transfer-controls`
- Deliverables: `deliv-phase16-transfer`
- Acceptance tests: `test-shared-controls-explicit`
- Anchor coverage: `ref-hg1223-quench`, `ref-bst-pqp`, `ref-fese-pqp`, Phase 15 outputs
- Forbidden proxies: `fp-transfer-without-analog-controls`

**Success Criteria**

1. `Hg1223`, `BST`, and `FeSe` appear on a common `PQP` parameter grid
2. Shared controls such as `TQ`, thermal budget, and retrieval handling are separated cleanly from route-specific ones
3. The missing-control ledger ranks the most important unresolved variables for `Hg1223`
4. The repo states which control variables should be prioritized before any broader route comparison is reopened

**Backtracking trigger:** If the analog set fails to define transferable controls, Phase 17 must narrow to a strictly `Hg1223`-only replication campaign with lower confidence.

**Plans:** 3/3 plans complete

**Executed outcome (2026-03-29):** The repo now carries a common `PQP` transfer table across `Hg1223`, `BST`, and `FeSe`, a shared-versus-route-specific control map, and a ranked `Hg1223` missing-control ledger. The strongest shared controls are lower `TQ`, post-quench thermal budget, and retrieval handling. The main remaining `Hg1223` gaps are exact `vQ`, handling disturbance, and sample-state dependence.

### Phase 17: Hg1223 Experiment-Facing Reproducibility Campaign

**Goal:** Design the smallest decisive experiment set that could tell whether `Hg1223` has a real reproducibility basin or just a narrow benchmark window.
**Depends on:** Phase 15 and Phase 16
**Requirements:** `CAMP-01`, `CAMP-02`, `CAMP-03`
**Contract Coverage:**
- Advances: `claim-reproducibility-campaign`
- Deliverables: `deliv-phase17-sweep`
- Acceptance tests: `test-campaign-decisive`
- Anchor coverage: Phase 15 and 16 outputs, `ref-phase14-decision`
- Forbidden proxies: `fp-campaign-without-missing-fields`

**Success Criteria**

1. The campaign sweeps the main carried variables: `PQ`, `TQ`, `vQ`, and sample state
2. In-DAC confirmation, thermal cycling, retrieval, and ex-DAC probes are sequenced explicitly
3. Replication and failure gates are numeric enough to reject false progress
4. The campaign can distinguish "headline reproduction" from "robust route"

**Backtracking trigger:** If the campaign cannot be made decisive because too many core variables remain hidden, Phase 18 must downgrade the `Hg1223` route.

**Plans:** 3/3 plans complete

**Executed outcome (2026-03-29):** The repo now defines a staged `29`-run `Hg1223` campaign that tests the benchmark window first, isolates `vQ` second, and stress-tests sample state and handling third. It also carries a stage-separated measurement sequence plus explicit headline-reproduction, basin, downgrade, and stop gates.

### Phase 18: v4.0 Route Update and Next-Step Memo

**Goal:** Decide whether `Hg1223` remains the primary route after the protocol audit, and emit the next-step memo without hiding the remaining gap to room temperature.
**Depends on:** Phase 15, Phase 16, and Phase 17
**Requirements:** `VALD-02`, `VALD-03`
**Contract Coverage:**
- Advances: `claim-route-update`
- Deliverables: `deliv-phase18-final-memo`
- Acceptance tests: `test-route-update-honest`
- Anchor coverage: all carried milestone outputs plus `ref-phase14-decision`
- Forbidden proxies: `fp-consumer-language-without-gap`

**Success Criteria**

1. The route memo states whether `Hg1223` got stronger or weaker after the protocol audit
2. The memo preserves the nickelate backup if `Hg1223` remains fragile
3. The next-step experiment set is explicit
4. Consumer-hardware language remains blocked unless the evidence changes drastically

**Backtracking trigger:** If the protocol audit leaves `Hg1223` too narrow or too opaque, the route memo must say so and lower the route's confidence explicitly.

**Plans:** 3/3 plans complete

**Executed outcome (2026-03-29):** `v4.0` closes with `Hg1223` still primary and bilayer nickelates still preserved as the backup route. The route is upgraded from `benchmark-solid but protocol-opaque` to `protocol-specified and campaign-defined, but still experimentally under-controlled`. The practical guardrail is unchanged: the best carried retained benchmark remains `151 K`, which is still `149 K` below room temperature.
