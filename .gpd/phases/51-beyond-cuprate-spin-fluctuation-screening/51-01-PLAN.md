# Phase 51: Beyond-Cuprate Spin-Fluctuation Screening

## Objective

Screen 5 non-cuprate material families for spin-fluctuation pairing strength, identifying any with lambda_sf > 3.5 that could credibly exceed Hg1223 on the path to 300 K.

## Context

- @data/hg1223/spin_susceptibility/pairing_results.json (cuprate baseline: lambda_sf = 1.8 single-site)
- @data/hg1223/spin_susceptibility/cluster/chi_cluster_results.json (cluster: lambda_sf = 2.88)
- @data/nickelate/rpa_lambda_sf_results.json (nickelate RPA)
- @scripts/nickelate_rpa/rpa_susceptibility.py (RPA framework to reuse)

## Tasks

### Task 1: Multi-family tight-binding + Lindhard screening code
type="auto"

Build a generalized screening script that computes chi_0(q) from literature tight-binding parameters and RPA lambda_sf for each candidate family:
1. Infinite-layer nickelates (NdNiO2) under >3% compressive strain
2. High-entropy transition-metal oxides (La,Nd,Sm,Gd,Y)NiO3
3. Pressurized Sr2RuO4 at ~1-2 GPa
4. Iron-based 1111 (LaFeAsO) under strain
5. Bilayer nickelate La3Ni2O7 under >3% strain + Sm substitution

For each: define tight-binding, compute chi_0, compute RPA chi, extract lambda_sf in the leading attractive channel.

### Task 2: Run screening and produce comparison table
type="auto"

Execute the screening code, validate chi_0 against known nesting features, and produce the comparison table:
family | lambda_sf | peak q-vector | pairing symmetry | operating conditions | 300 K plausibility

### Task 3: Assessment and ranking
type="auto"

Rank all families against the Hg1223 baseline (lambda_sf_cluster = 2.88). Determine whether any family exceeds lambda_sf = 3.5. Document 300 K plausibility for each.

## Verification

- Dimensional consistency: all chi in states/eV, lambda_sf dimensionless
- Nesting features match known literature for each family
- Cuprate baseline reproduced as sanity check
- lambda_sf values physically reasonable (0.01-5.0 range)

## Success Criteria

1. At least 5 families screened with chi_0 and lambda_sf
2. Comparison table produced with all required columns
3. Any lambda_sf > 3.5 family flagged as "cuprate-exceeding"
4. Honest assessment: most families expected below cuprates
