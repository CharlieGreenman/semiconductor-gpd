  <action>
    Build the reproduction protocol by layering three sources onto a single document:

    **Layer 1 -- Hg1223-specific parameters from arXiv:2603.12437:**
    - PQ window: 10.1, 18.9, 28.4 GPa (extract exact values)
    - TQ: 4.2 K (low-quench) and 77 K (warm-quench)
    - Retained Tc: 147-151 K zero-resistance at low TQ; ~139 K at warm TQ (PQ = 26 GPa)
    - Stability: >= 3 days at 77 K; deterioration at ~170-200 K
    - Sample size: 50-80 microns
    - Retrieved SC volume fraction: ~78% with partial annealing to ~140 K
    - Structural: tetragonal crystal structure retained post-PQP (synchrotron XRD)
    - Known difficulty: electrical leads breaking during pressure quench
    - PQP uplift: Delta_Tc = 151 - 133 = 18 K above stable ambient Tc

    **Layer 2 -- Transferable PQP methodology from FeSe (2021) and BST (2025):**
    - General DAC quench procedure: pressurize to PQ, cool to TQ, rapid decompression
    - Stability testing protocol: hold at cryogenic temperature, measure at intervals
    - FeSe reference: 37 K retained Tc, stable 7 days at 77 K, annealed at 300 K
    - BST reference: 10.2 K retained Tc, survived room temperature cycling
    - Key contrast: Hg1223 is MORE fragile than BST (degrades at 200 K vs survives 300 K)
    - Transferable lesson: vQ logging was implicit in prior PQP work but never published numerically

    **Layer 3 -- Phase 19 Stage A runbook skeleton:**
    - Map the 6-node matrix (A-01 through A-06) directly into the protocol
    - Incorporate all 6 preconditions from Phase 19
    - Incorporate the 9-step per-run protocol from Phase 19
    - Incorporate handling classes H0-H3 from Phase 19
    - Incorporate run-invalidation rules (6 rules) from Phase 19
    - Incorporate decisive-run standard from Phase 19

    **Protocol structure (sections):**
    1. Scope and purpose (independent reproduction, not original-group extension)
    2. Fixed node matrix (A-01 to A-06 with PQ/TQ/replicates, allow +/- 1 GPa tolerance on PQ)
    3. Success gate definition: retained ambient zero-resistance Tc >= 131 K (= 151 - 20); gap = 300 - 151 = 149 K explicit
    4. Minimum characterization suite:
       - Resistivity: 4-probe, zero-resistance criterion (R below noise floor), current density specified
       - Meissner: diamagnetic susceptibility, FC vs ZFC, minimum SC volume fraction threshold (>30% for T1, >50% for T2)
       - Structural: XRD (synchrotron preferred, lab source acceptable with caveats)
    5. Critical gap: vQ (decompression rate)
       - Flag as unpublished
       - Recommend: (a) request from original group, or (b) bracket by testing at least 2 decompression rates
       - Require: full pressure-release trace logging for every run regardless
    6. Preconditions before any run counts (6 items from Phase 19)
    7. Per-run protocol (9-step sequence from Phase 19, adapted for independent group)
    8. Handling classes (H0-H3 from Phase 19)
    9. Run invalidation rules (6 rules from Phase 19)
    10. Decisive-run standard (from Phase 19)
    11. Pressure calibration: specify functional requirement (ruby fluorescence or diamond Raman)
    12. Timeline: 6-month window per Phase 23 memo
    13. Protocol uncertainty section: list what is known, what is inferred from FeSe/BST, and what is genuinely unknown

    **VALD-01 enforcement:** Every Tc value in the protocol must label: (a) zero-resistance vs onset, (b) operating conditions (retained ambient, under pressure, metastable/stable).

    **VALD-02 enforcement:** The 149 K gap (300 - 151) must appear in the protocol with the guardrail statement: nothing in this protocol counts as room-temperature progress.

    **Forbidden:** Do NOT produce a protocol that omits vQ logging. Do NOT use onset for the success gate. Do NOT use room-temperature language.

    Also produce a machine-readable JSON version with the same content structured as: nodes array, success_gate object, characterization object, critical_gaps array, preconditions array, per_run_steps array, handling_classes array, invalidation_rules array.

  <name>Task 2: Build sample-state and handling-control checklist</name>
  <files>

  <action>
    Build a checklist that ensures every run produces diagnostic data whether it succeeds or fails. The checklist is completed per-run by the reproducing group. It must make vQ, thermal budget, and retrieval disturbance first-class logged variables.

    **Checklist sections:**

    1. **Pre-run sample state** (completed before any pressurization):
       - Sample ID (unique)
       - Sample class (source, batch)
       - Oxygen history class (delta value if measured, annealing history)
       - Geometry (dimensions, mass estimate)
       - Ambient Tc baseline measurement (before PQP -- should be ~133 K for Hg1223)
       - Contact configuration (4-probe geometry, lead material)

    2. **Pre-quench state** (completed under pressure before release):
       - Condition ID (A-01 through A-06)
       - Assigned handling class (H0-H3)
       - PQ achieved (GPa, calibration method: ruby fluorescence or diamond Raman)
       - TQ achieved (K, thermometry method)
       - Source-state confirmation: R(T) under pressure showing high-Tc state

    3. **Quench event** (completed during decompression):
       - Quench timestamp (ISO 8601)
       - Full pressure-release trace (P vs t, attached as file reference)
       - vQ estimate (GPa/s if derivable from trace; "not derivable" with reason if not)
       - Any anomalies (lead breakage, gasket failure, pressure cell issues)

    4. **Post-quench measurement** (completed after ambient reached):
       - First measurement timestamp (ISO 8601)
       - Thermal path from quench to first measurement (max temperature reached)
       - First ambient cryogenic R(T) (attached as file reference)
       - Retained Tc value (zero-resistance, K)
       - Retained Tc onset value (onset, K, for information only -- NOT for gate evaluation)
       - 77 K hold checkpoints (24h, 72h)

    5. **Warm-side testing** (if applicable per handling class):
       - 160-170 K checkpoint (H1+)
       - 200 K checkpoint (H2+)
       - 293 K checkpoint (H3 only)
       - Tc after each warm checkpoint

    6. **Retrieval and ex-DAC** (if applicable):
       - Retrieval thermal path (logged)
       - Ex-DAC transport R(T)
       - Meissner verification (FC/ZFC susceptibility, SC volume fraction)
       - XRD (structure confirmation)

    7. **Run validity assessment** (completed after all data collected):
       - Cross-check against 6 Phase 19 invalidation rules
       - Assign evidence tier: T0 (invalid), T1 (headline candidate), T2 (basin candidate), T3 (strengthened route)
       - Flag any missing logs

    **Each checklist field must be marked as:**
    - REQUIRED (run is T0 without it)
    - CONDITIONAL (required only for specific handling classes or stages)
    - OPTIONAL (informational, does not affect tier)

    **Cross-reference with Phase 19 invalidation rules:**
    Map each of the 6 invalidation rules to the specific checklist field(s) whose absence triggers it:
    1. Source state not confirmed -> Pre-quench: source-state confirmation
    2. vQ trace missing -> Quench event: pressure-release trace
    3. Sample class missing -> Pre-run: sample class + oxygen history
    4. Unplanned warm excursion -> Post-quench: thermal path field
    5. Retrieval without logged pathway -> Retrieval: thermal path
    6. Incomplete stage tags -> Run validity: all stage-tagged fields

    Also produce a machine-readable JSON version with fields array (each field: name, section, required/conditional/optional, invalidation_rule_ref if applicable, data_type).
  </action>
  <verify>
    1. Every Phase 19 invalidation rule (6 rules) maps to at least one checklist field
    2. vQ appears as REQUIRED (not optional) in the quench-event section
    3. Thermal path logging appears as REQUIRED for all handling classes
    4. Oxygen history appears as a logged variable (per Phase 20 sample-state-dependence failure mode)
    5. Evidence-tier assignment section references Phase 20 T0-T3 definitions
    6. Zero-resistance vs onset are separate fields (onset for information, zero-resistance for gate)
    7. The checklist is completable before any data is analyzed (it is a logging form, not an analysis form)
    8. JSON fields match markdown fields one-to-one
    9. VALD-01: the baseline Tc field specifies "zero-resistance, ambient, stable phase"
    10. No field in the checklist references unpublished vQ values -- only requires trace logging
  </verify>
