  <files>

  <action>
    Build a pre-registered route-confidence update map. Use a lookup-table approach (not continuous Bayesian updates) per the RESEARCH.md recommendation -- the decision space is discrete and small enough that a lookup table is superior.

    **Step 1: Enumerate all outcome classes.**
    The outcome taxonomy comes from Phase 20's 8 failure modes:
    1. target-state failure
    2. quench-trajectory failure
    3. sample-state dependence
    4. cryogenic-retention weakness
    5. warm-side fragility
    6. retrieval-induced loss
    7. onset-only ambiguity
    8. invalid / non-countable run

    Plus 3 success classes (not in Phase 20 because Phase 20 focused on failure):
    9. headline reproduction match (retained Tc >= 131 K, zero-resistance)
    10. headline exceed (retained Tc > 151 K -- unexpected, requires careful verification)
    11. partial retention (Tc_retained between stable ambient ~133 K and 131 K gate)

    **Step 2: For each outcome class, define the route-confidence update.**

    Structure each entry as:
    - Outcome class name
    - Evidence tier range (which tiers can this outcome achieve?)
    - What it means for route confidence
    - Route decision verb: one of {keep-primary, hold-pending-replication, hold-pending-diagnostic, downgrade-to-secondary, activate-pivot-trigger, no-route-update}
    - Gap update arithmetic (300 - Tc_retained = gap, or "unchanged" if the outcome does not produce a Tc measurement)
    - Next justified action (from Phase 20 routing tree, extended)
    - What this outcome does NOT mean (from Phase 20 failure mode map)

    **Step 3: Build the aggregate decision rules.**

    Define what happens after the FULL Stage A campaign (not just one run):
    - If >= 1 node achieves headline match (>= 131 K) at T1 or higher: keep Hg1223 primary, proceed to T2 replication at that node
    - If all 6 nodes are clean misses (valid logs, Tc < 131 K or no retention): this is strong evidence for non-reproducibility. Activate Phase 23 pivot trigger. Gap widens to 300 - 133 = 167 K (using stable ambient baseline).
    - If first 3 runs are all T0 (invalid): operational failure, troubleshoot before continuing. Do NOT update route confidence.
    - If mixed results (some hits, some misses): hold-pending-diagnostic. Classify each node individually. Look for PQ-dependence or TQ-dependence patterns.

    **Step 4: Include the pivot trigger from Phase 23.**

    Per Phase 23 next-step memo:
    - PQP reproduction fails -> fall back to stable ~134 K ambient Tc
    - Gap widens to 300 - 134 = 166 K (or 300 - 133 = 167 K; use the more conservative 134 K from Phase 23)
    - Evaluate nickelate promotion to co-primary per promotion trigger (100 K ambient zero-resistance)
    - If PQP fundamentally non-reproducible: escalate to co-primary program with nickelates

    **Step 5: Enforce VALD-01 and VALD-02.**
    - Every Tc value in the map must label operating state and Tc definition
    - The 149 K gap must appear with the guardrail: even a headline match is still 149 K below room temperature
    - Gap update for headline match: gap = 300 - Tc_retained (still ~149 K)
    - Gap update for partial retention: gap = 300 - Tc_retained (e.g., 300 - 140 = 160 K)
    - Gap update for no retention: gap = 300 - 134 = 166 K (stable ambient baseline)

    **Step 6: Include a "default / unknown outcome" row.**
    For any outcome not matching the 11 pre-defined classes: hold-pending-classification, no route update, escalate to researcher for classification before updating confidence.

    **Forbidden:** Do NOT produce entries where the same outcome triggers both upgrade and downgrade. Do NOT allow T0 outcomes to update route confidence. Do NOT use vague decision language ("reassess", "consider", "evaluate further" without specific criteria).

    Also produce a machine-readable JSON version with: outcome_classes array (each: name, tier_range, route_decision, gap_update, next_action), aggregate_rules object, pivot_trigger object.
  </action>
  <verify>
    1. All 8 Phase 20 failure modes have entries (cross-check by name)
    2. All 3 success classes have entries (headline match, headline exceed, partial retention)
    3. Default/unknown row exists
    4. No outcome maps to both upgrade and downgrade (logical consistency)
    5. All T0 entries say "no-route-update" (T0 is uninformative)
    6. Pivot trigger from Phase 23 appears with correct arithmetic (131 K gate, 166 K fallback gap)
    7. Gap arithmetic: 300 - 151 = 149, 300 - 134 = 166, all correct
    8. VALD-01: every Tc labels operating state and definition
    9. VALD-02: 149 K gap appears; guardrail statement present
    10. Route decisions use pre-defined verbs only (no vague language)
    11. Aggregate rules cover: all-hit, all-miss, all-T0, mixed scenarios
    12. JSON matches markdown (same outcome count, same decisions, same gap values)
  </verify>
